# Start a Business

1. Establish skills, experience, interests, constraints, budget, available time, and risk tolerance. Ask if not provided — don't assume.
2. Use `discover_business_opportunities` (EarnedLab MCP) or reason from the inputs to produce a short list of 3–4 realistic business directions. Never produce an unbounded brainstorm.
3. For each direction: explain fit, state key assumptions, identify the top risks, and suggest the first validation experiment.
4. Help the user compare options and select one. Do not select for them.
5. Once a direction is chosen, define: target customer, customer problem, offer, pricing model, and the first concrete validation test.
6. Create an EarnedLab workspace only with explicit user approval. Use `create_earnedlab_workspace` with the approved context.
7. Never promise income, revenue, or business success. Label all projections and estimates as estimates.
